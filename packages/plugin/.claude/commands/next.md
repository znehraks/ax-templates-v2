# /next

다음 스테이지로 전환합니다.

## 사용법

```
/next [--skip-validation] [-y|--yes]
```

## 옵션

- `--skip-validation`: 산출물 검증 건너뛰기
- `-y, --yes`: 확인 프롬프트 건너뛰기

## 동작

1. 현재 스테이지 확인
2. 현재 스테이지 산출물 검증
3. HANDOFF.md 존재 확인
4. 현재 스테이지를 `completed`로 표시
5. 다음 스테이지 확인 및 시작

## 전환 조건

다음 스테이지로 전환하려면:

1. ✅ 현재 스테이지의 모든 필수 산출물이 생성되어야 함
2. ✅ HANDOFF.md가 존재해야 함
3. ✅ (구현/리팩토링 스테이지) 체크포인트가 생성되어야 함

## 산출물 누락 시

산출물이 누락된 경우 경고가 표시되고 확인을 요청합니다:

```
⚠️  일부 산출물이 누락되었습니다:
  - outputs/architecture.md
  - outputs/tech_stack.md

산출물이 누락되었습니다. 그래도 진행하시겠습니까? (y/N)
```

## HANDOFF.md 누락 시

```
⚠️  HANDOFF.md가 생성되지 않았습니다.
생성: /handoff

HANDOFF.md 없이 진행하시겠습니까? (y/N)
```

## 모든 스테이지 완료 시

```
🎉 모든 파이프라인 스테이지가 완료되었습니다!
상태 확인: /status
```
