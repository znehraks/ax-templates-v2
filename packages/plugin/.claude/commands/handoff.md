# /handoff

현재 스테이지의 HANDOFF.md를 생성합니다.

## 사용법

```
/handoff [stage-id] [-i|--interactive] [-f|--force]
```

## 인자

- `stage-id` (선택): 특정 스테이지 지정 (기본: 현재 스테이지)
- `-i, --interactive`: 대화형 모드로 상세 정보 입력
- `-f, --force`: 기존 HANDOFF.md 덮어쓰기

## HANDOFF.md 구조

```markdown
# HANDOFF - [스테이지명]

## 생성 시간
{{timestamp}}

## 완료된 작업
- [x] 산출물 A 생성
- [x] 산출물 B 생성
- [x] ...

## 핵심 결정사항
- 결정 1: 이유
- 결정 2: 이유

## 성공한 접근법
- 접근법 1

## 실패한 접근법
- 접근법 1: 실패 이유

## 다음 단계 작업
1. 즉시 실행할 작업 1
2. 즉시 실행할 작업 2

## AI 호출 기록
| AI | 호출 시간 | 프롬프트 | 결과 | 상태 |
|----|----------|---------|------|------|
| Gemini | 14:30 | prompts/ideation.md | outputs/ideas.md | 성공 |

## 체크포인트
ID: cp_20240115_143000
복구: /restore cp_20240115_143000
```

## 대화형 모드

`-i` 옵션을 사용하면 각 섹션을 직접 입력할 수 있습니다:

```
/handoff -i
```

1. 완료된 작업 목록 입력 (에디터)
2. 핵심 결정사항 입력
3. 성공한 접근법 입력
4. 실패한 접근법 입력
5. 다음 단계 작업 입력
6. 체크포인트 참조 입력

## 자동 모드

기본 모드에서는 자동으로 기본 내용이 생성됩니다:

- 완료된 작업: 생성된 산출물 목록
- AI 호출 기록: 로그에서 자동 추출
- 다음 단계: 다음 스테이지 산출물 목록
